# Risk Framework

Use this file to define how discretionary trades should be sized, invalidated,
and reviewed.

## Core Rules

- Risk should be explicit before entry.
- Position size should follow a stated capital or portfolio-risk budget.
- Stops may be structural, volatility-based, or scenario-based, but must be
  named clearly.
- Good process matters more than short-term PnL.

## Review Questions

- Was the downside sized appropriately?
- Was the invalidation level real or arbitrary?
- Did the trade earn the right to be large?
- Did management improve or worsen the original setup?
